# Bokuchi Editor 使用指南

本指南介紹 Bokuchi Editor 的各種便利使用方式。

## 分頁功能使用方式

### 多檔案編輯
- **新增檔案**：`Ctrl+N`（Windows/Linux）或 `Cmd+N`（Mac）
- **開啟檔案**：`Ctrl+O`（Windows/Linux）或 `Cmd+O`（Mac）
- **切換分頁**：`Ctrl+Tab` 切換到下一個分頁，`Ctrl+Shift+Tab` 切換到上一個分頁

### 實際應用範例
1. 編輯主要文件
2. 在另一個分頁中開啟參考資料
3. 在另一個分頁中管理範本檔案
4. 在多個分頁之間複製貼上

## 縮放功能

### 縮放調整
- **放大**：`Ctrl++`（Windows/Linux）或 `Cmd++`（Mac）
- **縮小**：`Ctrl+-`（Windows/Linux）或 `Cmd+-`（Mac）
- **重設**：`Ctrl+0`（Windows/Linux）或 `Cmd+0`（Mac）

### 使用情境
- **簡報展示**：放大文字以提高可見度
- **精細編輯**：縮小文字以綜覽全局
- **雙螢幕**：調整至適合單一螢幕的大小

## 佈景主題功能

### 佈景主題切換
- **深色模式**：不易造成眼睛疲勞，適合長時間工作
- **淺色模式**：適合列印或在明亮環境下工作

### 自訂設定
- 可透過 CSS 變數進行細部自訂
- 也可以設定預覽的自訂 CSS

## 狀態持久化

### 自動儲存功能
- 應用程式重新啟動時會還原狀態
- 開啟的分頁、目前分頁和未儲存的內容都會被保留
- 檔案路徑也會被記住，因此外部修改的檔案會自動重新載入

### 使用小技巧
- 中斷工作時，直接關閉應用程式即可
- 下次啟動時會還原上次的工作狀態
- 同時處理多個專案時特別方便

## 變數功能應用

### 建立範本
```markdown
<!-- @var projectName: 我的專案 -->
<!-- @var version: 1.0.0 -->
<!-- @var author: 你的名字 -->

# {{projectName}} 說明文件

版本：{{version}}
作者：{{author}}

## 概述
本文件描述 {{projectName}} 專案。
```

### 全域變數的使用
- 將公司名稱、部門名稱、作者名稱等共用資訊設定為全域變數
- 在多份文件中管理一致的資訊

## 核取方塊功能使用方式

### 任務管理
```markdown
## 今日待辦事項
- [ ] 準備會議資料
- [ ] 進行程式碼審查
- [ ] 更新說明文件
- [x] 確認早上的電子郵件
```

### 專案管理
- 視覺化進度狀態
- 確認已完成的任務
- 在團隊中分享進度

## 檔案操作最佳實踐

### 檔案命名規範
- 包含日期：`2025-01-15-meeting-notes.md`
- 包含專案名稱：`project-alpha-spec.md`
- 版本控制：`api-docs-v2.md`

### 資料夾結構
```
docs/
├── meetings/
│   ├── 2025-01-15-weekly.md
│   └── 2025-01-22-weekly.md
├── specs/
│   ├── api-spec.md
│   └── ui-spec.md
└── templates/
    ├── meeting-template.md
    └── project-template.md
```

## 總結

Bokuchi Editor 不僅僅是一個 Markdown 編輯器，更提供了高效率的文件建立環境。
它在以下方面表現尤為出色：

- **多工支援**：同時編輯多個檔案
- **易用性**：直覺操作與狀態持久化
- **可自訂性**：透過佈景主題和變數進行靈活設定
- **實用性**：核取方塊和縮放等實用功能

結合這些功能，可以實現更加高效且舒適的文件建立。
